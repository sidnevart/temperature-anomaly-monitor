# Модель и правила обнаружения

Обнаружение — два слоя: правила (объяснимые) + Isolation Forest (ИИ).

## 1. Правила (rule-based)

Реализованы в `anomaly_detection.detect_anomalies()`, пороги — в `RULE_PARAMS`.

| Событие | Условие | Уровень |
|---|---|---|
| Потеря сигнала | `is_missing == 1` | Medium |
| Резкий скачок температуры | `abs_temp_diff > sharp_jump_diff` (5 °C) | High |
| Сильное отклонение от нормы | `abs_z_score > z_score` (3) | Warning |
| Зависание датчика | `is_stuck == 1` (точное равенство подряд ≥10) | Medium |
| Отклонение от группы датчиков | `abs_diff_from_group_mean > group_deviation` (8 °C) | Warning |
| Устойчивый перегрев | наклон `rolling_mean` по окну 20 > `overheat_slope` (0.12) | Warning |

`is_stuck` считается в `preprocessing.py` через **точное равенство** подряд
`STUCK_MIN_RUN` значений (а не порог `|Δt|<0.05`). Это важно: пороговый
критерий ловил ступени квантования АЦП реальных данных как «зависание»
(тысячи ложных тревог).

`abs_z_score` считается по **скользящему** `rolling_mean/rolling_std`
(локальный z-score), а не по глобальному среднему датчика. Глобальный на
растущем реальном сигнале ловил весь верхний полубас как аномалию.

## 2. Isolation Forest

- Обучается **только на `scenario == 'normal'`** (`train_model.train()`),
  чтобы учить штатный режим, а аномалии становились «нетипичными».
- Сохраняется: `models/scaler.joblib`, `models/iforest.joblib`,
  `models/model_meta.json`.
- `detect_anomalies()` загружает сохранённую модель (`transform`/`predict`,
  без `fit`). Если модели нет — обучается на лету (запасной режим с data
  leakage и более низкой точностью).
- `contamination` (доля аномалий) по умолчанию 0.04.

### Аномалия только по ИИ

Если Isolation Forest помечает точку как аномальную, а правила — нет, ей
присваивается тип «Нетипичное поведение по ИИ-модели». Это полезно для ловли
тонких отклонений, которые не описаны правилами (например, медленный дрейф
датчика — его наклон ниже пика нормальной синусоиды, правилом от нормы не
отделить, но ИИ ловит).

## 3. Объединение

```
final_anomaly = (rule_anomaly == 1) | (iforest_anomaly == 1)
```

Уровень риска уточняется по `anomaly_score_norm`:
- > 0.85 → High;
- 0.60–0.85 → Medium (если ещё не выше).

## 4. Метрики (бенчмарк)

На синтетике с разметкой (до/после правок точности):

| Метрика | Было | Стало |
|---|---|---|
| F1 | 0.48 | ~0.80 (тест) / 0.66 (полные данные) |
| Precision | 0.56 | 0.74 |
| FP на normal | 8.1% | 5.3% |

На реальных `Т2.csv` (после объединения с `mod_AI_2`): 94% ложных тревог → 10%.
